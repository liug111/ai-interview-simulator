# 使用指南

## 快速开始

### 1. 配置 API Key

参考 `config_example.md` 配置环境变量。推荐使用阿里千问 DashScope。

### 2. 准备简历

创建一个文本文件或直接粘贴简历内容。参考格式：

```text
姓名：张三
学历：本科，计算机科学与技术，XX大学（2026届毕业）
技能：Python、Java、MySQL、Redis、Docker
项目：在线商城系统（Spring Boot + Vue）
实习：XX科技有限公司 - 后端开发实习生（3个月）
```

### 3. 运行模拟面试

```bash
python scripts/interview_simulator.py -p "Java后端开发" -r resume.txt -t "技术面"
```

### 4. 面试流程

```
┌─────────────────────────────────────────┐
│  AI 面试官提出第 1 题                    │
│  标记 【提问1】                          │
└────────────────┬────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────┐
│  你输入回答                              │
└────────────────┬────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────┐
│  AI 追问 / 深入                          │
│  标记 【追问1】                          │
└────────────────┬────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────┐
│  你再次回答                              │
└────────────────┬────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────┐
│  AI 给出评分 + 改进建议                  │
│  标记 【评分1】                          │
└────────────────┬────────────────────────┘
                 │
                 ▼
           进入下一题（提问2 → 追问2 → 评分2 → … ）
```

### 5. 结束面试

- 输入 `exit` — 立即退出
- 输入 `summary` — 跳过剩余题目，AI 给出整体总结评价

## 面试类型说明

| 类型 | 特点 | 适合场景 |
|------|------|---------|
| 技术面 | 追问技术细节、算法、架构设计 | 技术岗位面试准备 |
| HR面 | 考察软素质、职业规划、团队协作 | 综合面试、终面准备 |
| 压力面 | 不断质疑打断，测试抗压能力 | 高压岗位、群面准备 |
| 群面 | 给出小组讨论题目，点评表现 | 校招群面、无领导小组讨论 |

## 最佳实践

1. **准备越详细，效果越好**：简历越完整，AI 提出的问题越精准
2. **认真回答每个问题**：AI 会根据回答内容进行深度追问
3. **多类型交叉训练**：先用「技术面」练专业，再用「压力面」练心态
4. **关注评分反馈**：每题评分中的「扣分项」和「改进建议」是提升关键
5. **完整面试后复盘**：用 `summary` 获取整体评价，记录薄弱项

## 常见问题

**Q: 报错 `请设置环境变量`？**
A: 参考 `config_example.md` 配置 API Key 环境变量

**Q: 网络连接失败？**
A: 确保可以访问 API 地址。国内用户推荐使用阿里千问 DashScope

**Q: 回答太长了/太短了？**
A: 正常对话即可。如果 AI 没有追问直接评分，说明你的回答已经很完整
